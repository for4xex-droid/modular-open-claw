# 🌌 Antigravity CodeWiki 活用ガイド

Antigravityプロジェクトにおける「爆速開発」と「品質維持」を両立させるための、CodeWiki活用エコシステムについて解説します。

## 🚀 活用を加速させる 3 つの仕組み

本プロジェクトでは、CodeWikiを単なる「置き場」にせず、開発フローの各フェーズで自然に活用できるよう調整されています。

### 1. 自動「執事」パイプライン (CI/CD 連携)
実装を完了し、`git push` した瞬間に、CodeWikiがあなたの代わりにドキュメントを書き換えます。
- **仕組み**: GitHub Actions (`update-docs.yml`) がコードの差分を検知し、AIが最新の仕様、DBスキーマ、APIエンドポイントの解説を更新。
- **メリット**: 「ドキュメントが古い」というストレスがこの世から消滅します。

### 2. 管理画面への直埋め込み (Live Dashboard)
Antigravityの管理画面（`localhost:3000`）には、CodeWiki Cloudから最新の「API利用ガイド」や「ビジネスロジック解説」をリアルタイムで取得する機能が統合されています。
- **仕組み**: 専用APIエンドポイントを通じて、ブラウザ上で常に最新の構造図（Mermaid図）やシーケンス図を確認可能。
- **メリット**: エディタとブラウザを行き来するだけで、システムの全貌を把握できます。

### 3. Vibe Coding を支える「構造検索」
Antigravityのような大規模なボイラープレートで迷子にならないための仕組みです。
- **仕組み**: AIが「どのファイルが何に責任を持っているか」を完全に理解。
- **活用例**: 
  - 「課金フローのシーケンス図を出して」
  - 「新しくモデルを追加したいけど、既存のどこを参考にすればいい？」
  - 「このリファクタリングの影響範囲をリストアップして」

## 🛠️ 開発者向けクイックスタート（3ステップ）

### Step 1: 実装する
`libs/core` や `libs/infrastructure` で機能を実装します。

### Step 2: Push する
```bash
git add .
git commit -m "feat: 新しい課金ロジックを追加 (#123)"
git push origin main
```
この時、背後でGitHub Actionsが走り、CodeWikiが情報を同期します。

### Step 3: 管理画面で確認する
`http://localhost:3000` にアクセスし、更新されたドキュメントや図解を確認。チームメンバーや未来の自分に、変更の意図が正しく伝わっているかチェックします。

---

## 💎 運用ルール（Iron Principles）

1. **ドキュメントは「書く」のではなく「生成させる」**
   READMEに長文を書く必要はありません。コードに適切なコメント（Doc Comments）を残せば、CodeWikiがそれを読み取って最高のWikiに仕上げます。
2. **迷ったら「聞く」**
   構造がわからなくなったらCodeWikiのチャット機能を使いましょう。彼はAntigravityの全構造を知り尽くしています。
3. **常に「True」である状態を保つ**
   デプロイパイプラインをバイパスせず、常にCI経由でドキュメントを最新化してください。
