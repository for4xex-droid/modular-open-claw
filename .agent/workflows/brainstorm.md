# /brainstorm - 3人の賢者による会議

単一の解決策に飛びつくのではなく、3つの異なるアプローチをシミュレーションし、最良の解を導き出します。
アーキテクチャ選定、リファクタリング方針の決定、複雑なバグの原因究明に有効です。

## 実行プロセス

**Sequential Thinking** を使用して、以下のプロセスを展開してください。

### Phase 1: 発散 (Divergence) 🌿
課題に対して、**方向性の異なる3つの解決案**を提示してください。
似たり寄ったりの案は避けてください。

- **Plan A (堅実)**: 既存パターン踏襲、リスク最小、保守性重視。
- **Plan B (革新)**: パフォーマンス重視、新技術採用、ドラスティックな変更。
- **Plan C (別解)**: 視点を変えたアプローチ（例: そもそもコードを書かずに解決できないか？など）。

### Phase 2: シミュレーション (Simulation) 🎲
各プランについて、以下を思考実験します。

1. **実装コスト**: 何時間/何日かかるか？
2. **リスク**: 将来的にどんなバグや負債を生むか？
3. **効果**: 課題をどれけ解決できるか？

### Phase 3: 収束 (Convergence) 🎯
3つの案を比較評価し、**結論**を出します。
- 1つの案を採用するか、良いとこ取り（Plan D）を作る。
- なぜその案がベストなのか論理的に説明する。
